# Neo4j Semantic Query Service

A FastAPI service that ingests semantic API profiles produced by `banking_opa_profiler.py`, stores them as a Neo4j knowledge graph, and answers natural-language or structured questions about APIs.

## Capabilities

- Business capability, entity, workflow, lifecycle, and semantic meaning queries
- Risk, compliance, data sensitivity, and policy violation queries
- Consumer interaction queries
- Semantic duplicate detection
- API dependency traversal and blast-radius analysis
- Idempotent batched ingestion
- API-key and admin-key authentication
- Health probes and Prometheus metrics
- Parameterized server-owned Cypher only

## Start the stack

```bash
cp .env.example .env
```

Set secure values in `.env`:

```dotenv
NEO4J_PASSWORD=replace-with-a-strong-password
API_KEY=replace-reader-key
ADMIN_API_KEY=replace-admin-key
```

Start Neo4j and the API:

```bash
docker compose --env-file .env up --build -d
```

Health checks:

```bash
curl http://localhost:8080/health/live
curl http://localhost:8080/health/ready
```

FastAPI documentation is available at `http://localhost:8080/docs`. Neo4j Browser is available at `http://localhost:7474`.

## Ingest data

Load the bundled sample data:

```bash
docker compose exec semantic-query \
  python -m app.cli ingest \
  /data/sample-profiles.json \
  --source sample
```

Replace `data/banking-api-profiles.json` with the JSON generated by `banking_opa_profiler.py`, then validate it:

```bash
docker compose exec semantic-query \
  python -m app.cli ingest \
  /data/banking-api-profiles.json \
  --source banking-profiler \
  --dry-run
```

Ingest it:

```bash
docker compose exec semantic-query \
  python -m app.cli ingest \
  /data/banking-api-profiles.json \
  --source banking-profiler
```

## Ask semantic questions

```bash
curl -s http://localhost:8080/v1/query \
  -H "Content-Type: application/json" \
  -H "X-API-Key: replace-reader-key" \
  -d '{
    "question": "What business capability does payment-approval represent?"
  }' | python3 -m json.tool
```

Examples:

```text
What entities does payment-approval manipulate?
What workflow does payment-execution belong to?
What risk and security sensitivity does payment-approval have?
How do consumers interact with payment-approval?
What semantic meaning does payment-execution have?
Show public high-risk APIs that process PCI data.
Show semantic duplicate APIs.
What is impacted if payment-execution fails?
Show the dependency path between payment-approval and payment-execution.
```

## Structured search

```bash
curl -s http://localhost:8080/v1/search \
  -H "Content-Type: application/json" \
  -H "X-API-Key: replace-reader-key" \
  -d '{
    "exposures": ["Partner"],
    "min_risk": "High",
    "contains_pci": true,
    "limit": 100
  }' | python3 -m json.tool
```

## Dependency impact

```bash
curl -s \
  "http://localhost:8080/v1/graph/impact/payment-execution?direction=dependents&max_depth=5" \
  -H "X-API-Key: replace-reader-key" \
  | python3 -m json.tool
```

## Security

The service never accepts caller-supplied Cypher. Natural-language questions are converted to an allowlisted filter model and executed with parameterized server-owned queries. Use a secret manager for production credentials and restrict Bolt access so only the service can reach Neo4j.
